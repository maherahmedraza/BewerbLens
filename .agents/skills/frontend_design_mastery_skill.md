# Frontend Design Mastery Skill
## Agentic Capability: Visual Craft & Interface Architecture
**Version:** 1.0.0  
**Compatibility:** Universal (LLM-agnostic agent instruction)  
**Scope:** Frontend UI/UX generation, component architecture, and visual identity systems  
**Classification:** Creative Direction + Implementation Protocol

---

## 1. Core Philosophy: The Anti-Formula

You are not a template engine. You are a design intelligence tasked with producing **context-specific visual artifacts** that carry intention, restraint, and surprise. The goal is to create interfaces that feel **authored by a human designer with a point of view**, not generated by an optimization algorithm converging on median taste.

**Prime Directive:** Every design must feel like it could only exist for its specific context. No two outputs should share the same aesthetic DNA unless explicitly requested.

**The "AI Slop" Prohibition:**  
Never produce generic AI-generated aesthetics. This includes overused font families (Inter, Roboto, Arial, system-ui), cliched color schemes (particularly purple-to-blue gradients on white backgrounds), predictable card-based layouts with soft shadows, and cookie-cutter component patterns that lack context-specific character. If the design looks like it could be a Dribbble trending page from 2023, reject it and regenerate with a divergent constraint.

## 2. Typography: The Voice of the Interface

### 2.1 Principles
Typography is the primary carrier of personality. Choose fonts that are beautiful, unique, and interesting. Avoid generic fonts like Arial and Inter; opt instead for distinctive choices that elevate the frontend's aesthetics. Seek unexpected, characterful font choices. Pair a distinctive display font with a refined body font.

### 2.2 Strategic Pairing Framework
| Role | Selection Criteria | Examples (Use as archetypes, not defaults) |
|------|-------------------|-------------------------------------------|
| **Display / H1-H2** | High contrast, unique geometry, expressive | Editorial New, GT Super, Ogg, Fraunces, Kablammo, Shrikhand, Uncut Sans |
| **Body / UI Text** | Legible but not invisible, slight character | Suisse Int'l, Canela Deck, Lyon Text, PP Neue Montreal, Source Serif 4, iA Writer Quattro |
| **Mono / Technical** | Functional but distinctive | Berkeley Mono, GT America Mono, JetBrains Mono (only if customized), SF Mono |

### 2.3 Implementation Rules
- **Never use** `font-family: system-ui, -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, sans-serif` as a lazy fallback. If a custom font fails, the fallback should still be considered (e.g., `Georgia` for serif contexts, `Helvetica Now` for grotesque contexts).
- Use `font-feature-settings` and `font-variant-numeric` to unlock OpenType features (tabular nums, ligatures, stylistic alternates).
- Establish a **modular scale** (1.25 or 1.414) but break it intentionally at display sizes for impact.
- Line-height should be tight for display (0.9–1.0) and generous for body (1.5–1.7).
- Letter-spacing: negative for large display, slight positive for small caps/labels.

### 2.4 Anti-Patterns
- [ ] Using Inter or Roboto because "they're safe"
- [ ] Pairing two sans-serifs without clear hierarchy
- [ ] Ignoring font loading strategy (use `font-display: swap` with a curated fallback)
- [ ] More than 3 font families in one project

## 3. Color & Theme: Cohesive Aesthetic Systems

### 3.1 Principles
Commit to a cohesive aesthetic. Use CSS variables for consistency. Dominant colors with sharp accents outperform timid, evenly-distributed palettes. A strong theme has a point of view: it is either boldly restrained (monochrome + one electric accent) or deliberately maximalist (orchestrated clashing hues with unifying treatment).

### 3.2 Variable Architecture
```css
:root {
  /* Dominant: 70% of UI */
  --color-bg-primary: #0a0a0a;
  --color-bg-secondary: #141414;
  --color-text-primary: #f0f0f0;
  --color-text-secondary: #888888;

  /* Accent: 10-15%, high chroma */
  --color-accent: #ff3b00;
  --color-accent-muted: #ff3b0040;

  /* Utility: borders, dividers, subtle surfaces */
  --color-border: rgba(255, 255, 255, 0.08);
  --color-surface-elevated: #1a1a1a;
}
```

### 3.3 Palette Strategies (Rotate Between These)
1. **Chiaroscuro Digital**: Deep blacks + warm off-whites + single saturated accent (vermillion, acid green, electric cyan)
2. **Desaturated Luxury**: Warm greys, stone, taupe + gold or deep burgundy accent
3. **Hyper-Contemporary**: White/off-white base + neon or fluorescent accent with heavy black typography
4. **Atmospheric Depth**: Navy/deep indigo base + coral or peach gradient accents
5. **Organic Brutalism**: Raw concrete greys + oversaturated nature tones (forest green, clay orange)

### 3.4 Rules
- Never use default Tailwind colors unmodified. If using a framework, override the entire scale.
- Contrast ratios must meet WCAG AA, but AAA can be sacrificed for large display text if it serves the aesthetic (document this decision).
- Use HSL or OKLCH for color manipulation in CSS variables to enable programmatic theming.
- Dark mode should not be an afterthought; it should be the default or a fully authored alternate reality, not an inversion.

## 4. Motion: Orchestrated Time

### 4.1 Principles
Use animations for effects and micro-interactions. Prioritize CSS-only solutions for HTML. Use Motion library for React when available. Focus on high-impact moments: one well-orchestrated page load with staggered reveals (animation-delay) creates more delight than scattered micro-interactions. Use scroll-triggering and hover states that surprise.

### 4.2 Motion Taxonomy
| Category | Tool | Purpose |
|----------|------|---------|
| **Page Load** | CSS `@keyframes` + `animation-delay` stagger | Establish hierarchy and rhythm |
| **Scroll Reveal** | Intersection Observer + CSS classes | Progressive disclosure without jank |
| **Hover States** | CSS `transition` | Physicality and responsiveness |
| **Complex Sequences** | Framer Motion / GSAP | Layout transitions, shared element transitions |
| **Ambient** | CSS infinite loops | Subtle life (grain shift, slow gradient drift) |

### 4.3 Orchestration Guidelines
- **Stagger Formula:** Base delay × index. Use `calc(var(--base-delay) * var(--item-index))`.
- **Easing:** Never use `ease-in-out` for entrances. Use `cubic-bezier(0.16, 1, 0.3, 1)` (expo out) for entrances, `cubic-bezier(0.7, 0, 0.84, 0)` for exits.
- **Duration Scale:** Micro (150ms), Standard (300ms), Emphasis (500ms), Ambient (8s–20s).
- **Transform Priority:** Only animate `transform` and `opacity`. Never animate `width`, `height`, `top`, `left`, `margin`.
- **Reduced Motion:** All motion must respect `prefers-reduced-motion: reduce` by defaulting to instant state changes or opacity-only transitions.

### 4.4 High-Impact Patterns
1. **The Curtain Reveal:** Content slides up from masked containers with slight overshoot.
2. **Typography Materialization:** Letters stagger in from `opacity: 0` and `translateY(20px)` with tight delays (20–40ms per character for display text only).
3. **Hover Displacement:** Elements shift slightly against cursor movement (parallax hover) or reveal hidden layers.
4. **Scroll Velocity Distortion:** Skew or scale elements based on scroll speed (subtle, max 5° skew).

## 5. Spatial Composition: Defying the Grid

### 5.1 Principles
Unexpected layouts. Asymmetry. Overlap. Diagonal flow. Grid-breaking elements. Generous negative space OR controlled density. The layout should feel composed, not assembled.

### 5.2 Composition Strategies
- **Asymmetric Balance:** Place the visual weight at 1/3 or 2/3 marks, never dead center.
- **Intentional Overlap:** Let text overlap images, cards overlap sections, or navigation overlap content. Use `mix-blend-mode: difference` or careful z-index layering.
- **Broken Grid:** Elements that escape their containers by 10–15% create tension and dynamism.
- **Diagonal Flow:** Use `clip-path`, rotated containers, or CSS `skew` to create directional energy.
- **Density Contrast:** Pair extremely sparse hero sections with information-dense lower sections, or vice versa. Never maintain uniform density throughout.

### 5.3 Layout Anti-Patterns
- [ ] Centered single-column text for entire pages
- [ ] Equal-width three-column feature grids
- [ ] Hero image on left, text on right (unless heavily subverted)
- [ ] Navigation bar + hero + 3 features + testimonial + CTA + footer (the startup template)
- [ ] Consistent padding throughout (vary section padding dramatically: 4rem vs 12rem)

## 6. Backgrounds & Visual Details: Atmospheric Depth

### 6.1 Principles
Create atmosphere and depth rather than defaulting to solid colors. Add contextual effects and textures that match the overall aesthetic. Apply creative forms like gradient meshes, noise textures, geometric patterns, layered transparencies, dramatic shadows, decorative borders, custom cursors, and grain overlays.

### 6.2 Texture & Effect Library
| Effect | Implementation | Best For |
|--------|---------------|----------|
| **Gradient Mesh** | CSS `radial-gradient` layers or WebGL | Hero backgrounds, dark themes |
| **Film Grain** | SVG noise filter or CSS `background-image` data-URI | Adding analog warmth to digital interfaces |
| **Geometric Patterns** | CSS `repeating-linear-gradient` or SVG patterns | Technical/editorial contexts |
| **Glassmorphism (Subtle)** | `backdrop-filter: blur(12px)` + semi-transparent border | Overlapping layers, not entire cards |
| **Dramatic Shadows** | Layered box-shadows with color tinting | Elevation without default grey shadows |
| **Custom Cursors** | SVG cursors or `cursor: none` + positioned element | Immersive experiences, galleries |
| **Border Treatments** | 1px hairlines with gradient fills, double borders, or inset shadows | Refinement and precision |

### 6.3 Atmospheric Rules
- Backgrounds should have **depth layers**: base color + subtle gradient + texture overlay at 3–5% opacity.
- Shadows should carry color. A warm interface uses warm shadows; a cold interface uses blue-tinted shadows.
- Use `pointer-events: none` on decorative layers to prevent interaction blocking.
- Grain should be animated slowly (shift `background-position` every 100ms) to prevent static banding.

## 7. Implementation Complexity Calibration

**Match implementation complexity to the aesthetic vision.**

- **Maximalist / Expressive / Experimental:** Elaborate code with extensive animations and effects. Multiple WebGL contexts, complex CSS architectures, custom cursors, scroll-jacking, shader backgrounds. This requires heavy CSS/JS investment and performance budgeting.
- **Minimalist / Refined / Editorial:** Restraint, precision, and careful attention to spacing, typography, and subtle details. Elegance comes from executing the vision well. Fewer animations, but each one is perfectly tuned. Generous whitespace. Precise alignment to the pixel.
- **Brutalist / Raw:** Intentionally unpolished code, visible grid lines, system-default form elements (subverted), high contrast, raw HTML energy. The complexity lies in the confidence of the restraint.

**Performance Budget:**
- First Contentful Paint < 1.5s
- Time to Interactive < 3.5s
- Cumulative Layout Shift < 0.1
- If heavy effects push beyond this, implement progressive enhancement (load effects after core content).

## 8. Creative Decision Framework

Before generating any interface, answer these questions to ensure divergence from generic output:

1. **What is the emotional temperature?** (Warm/inviting, Cold/precise, Energetic/chaotic, Contemplative/slow)
2. **What era of design does this belong to?** (1920s editorial, 1970s sci-fi, 1990s web, 2030s speculative, timeless craft)
3. **What is the one unexpected choice?** (A fluorescent accent in a legal document interface. A serif font in a dashboard. A horizontal scroll in a portfolio.)
4. **What is being celebrated?** (The content, the interaction, the negative space, the typography itself, the imagery)
5. **What is the dominant spatial relationship?** (Overlap, isolation, density, flow, fracture)

**Constraint Roulette (Apply at least one per project):**
- No border-radius on any element
- Only one color plus black and white
- All typography must be italic
- No images allowed — only color, type, and shape
- Every section must overlap the previous
- Navigation must be unconventional (bottom, vertical, hidden behind interaction, scattered)

## 9. Quality Assurance Checklist

Before finalizing any design output, verify:

- [ ] **Typography:** Are the fonts distinctive? Would a user notice them? Is the pairing intentional?
- [ ] **Color:** Is there a dominant color with sharp accents? Are CSS variables used consistently? Is the palette timid or bold?
- [ ] **Motion:** Is there a choreographed entrance? Do hover states surprise? Is `prefers-reduced-motion` handled?
- [ ] **Layout:** Is there asymmetry, overlap, or grid-breaking? Is the density varied?
- [ ] **Backgrounds:** Is there atmosphere beyond solid color? Are textures contextual?
- [ ] **Anti-Slop:** Does this look like it could be from a 2023 AI template? If yes, restart with a constraint from Section 8.
- [ ] **Complexity Match:** Does the code complexity match the aesthetic ambition?
- [ ] **Context Specificity:** Could this design work for any other random website? If yes, it is too generic. Add context-specific constraints.

## 10. Technology Stack Guidance

### HTML/CSS-Only Projects
- CSS custom properties for theming
- `@property` for typed custom properties (animations)
- CSS Grid with `subgrid` where supported
- Container queries for component-level responsiveness
- `clamp()` for fluid typography and spacing
- SVG filters for advanced effects (noise, displacement)

### React / Modern Framework Projects
- Framer Motion for layout animations and gestures
- CSS Modules or Tailwind (heavily customized) for styling
- GSAP + ScrollTrigger for complex scroll orchestration
- Lenis for smooth scrolling (when appropriate)
- React Spring for physics-based interactions

### Performance Guardrails
- Use `will-change` sparingly and remove after animation
- Implement `content-visibility: auto` for off-screen sections
- Lazy load below-fold images and heavy effects
- Use `transform3d` to force GPU compositing

## 11. Examples of Divergent Aesthetics

To demonstrate the expected variety, here are three radically different valid approaches:

### A. "The Archival Editor" (Light, Dense, Typographic)
- **Theme:** Warm off-white (#f4f1ea) background, near-black text (#1a1a1a), single accent of deep crimson (#c41e3a)
- **Fonts:** GT Super (display) + Suisse Int'l (body)
- **Layout:** Asymmetric two-column with heavy editorial margins. Text overlaps a faint, oversized background index number.
- **Motion:** Staggered text reveal on load. No scroll animations. Hover states invert colors instantly.
- **Details:** Hairline 1px borders, no shadows, paper-like texture overlay at 2%.

### B. "The Subterranean Interface" (Dark, Atmospheric, Maximal)
- **Theme:** Near-black (#050505) with gradient mesh in deep blues and purples (not the cliché bright purple). Accent is acid green (#39ff14).
- **Fonts:** Kablammo (display, used sparingly) + Berkeley Mono (body)
- **Layout:** Broken grid with cards floating at different z-depths. Diagonal section dividers.
- **Motion:** Grain shift, cursor-reactive gradient mesh, scroll-triggered skew on images, page load with 800ms stagger.
- **Details:** Glassmorphism panels, custom cursor, animated noise texture, dramatic colored shadows.

### C. "The Bureaucratic Brutalist" (Raw, Systemic, Confrontational)
- **Theme:** Pure white background, system-black text, single raw blue (#0000ff) for links/interactions only.
- **Fonts:** Times New Roman (display, intentionally raw) + Arial (body, intentionally generic but subverted by layout)
- **Layout:** Exposed grid lines. Elements touching edges with no padding. Navigation is a raw unordered list at the bottom of the viewport.
- **Motion:** None. Instant state changes. The brutality is in the stillness.
- **Details:** Visible table borders, raw form elements, no border-radius anywhere, ASCII dividers.

## 12. Final Directive

**Interpret creatively and make unexpected choices that feel genuinely designed for the context. No design should be the same. Vary between light and dark themes, different fonts, different aesthetics. NEVER converge on common choices (Space Grotesk, for example) across generations.**

If you find yourself defaulting to a safe choice, that is the signal to choose the dangerous one. Safe design is invisible. Dangerous design, when executed with precision, is unforgettable.

**Generate with conviction. Design with prejudice. Build with craft.**
