### Sprint 1 Tasks (Ordered by Execution)

#### Task 1: T-001 — Run migration_multiuser.sql (P0, 3 SP)

**Branch:** `feature/T-001-multiuser-migration`

**BLOCKER: Migration is NOT idempotent.** It was partially applied previously — `user_profiles` table and its RLS policies already exist. PostgreSQL's `CREATE POLICY` has no `IF NOT EXISTS` syntax, so re-running the migration fails at line 43.

**Implementation — Create idempotent migration:**
Create a new file `migration_multiuser_idempotent.sql` that wraps the original migration with safety checks:

1. **Tables** (lines 16-37, 69-82): Already use `CREATE TABLE IF NOT EXISTS` — safe to re-run
2. **RLS enable** (lines 40, 85, 174, 189, 200, 211): `ALTER TABLE ... ENABLE ROW LEVEL SECURITY` is idempotent — safe to re-run
3. **Policies** (lines 43-49, 87-89, 176-221): Must add `DROP POLICY IF EXISTS "policy_name" ON table;` before each `CREATE POLICY` statement. Affects 11 policies:
   - `"Users can view own profile"` on `user_profiles`
   - `"Users can update own profile"` on `user_profiles`
   - `"Users can manage own filters"` on `email_filters`
   - `"Users can view own applications"` on `applications`
   - `"Users can insert own applications"` on `applications`
   - `"Users can update own applications"` on `applications`
   - `"Users can view own emails"` on `raw_emails`
   - `"Users can insert own emails"` on `raw_emails`
   - `"Users can view own pipeline runs"` on `pipeline_runs`
   - `"Users can insert own pipeline runs"` on `pipeline_runs`
   - `"Users can view own pipeline logs"` on `pipeline_run_logs`
4. **Trigger** (lines 61-63): Must add `DROP TRIGGER IF EXISTS on_auth_user_created ON auth.users;` before `CREATE TRIGGER`
5. **Indexes** (lines 92-93, 165-167): Already use `CREATE INDEX IF NOT EXISTS` — safe to re-run
6. **Functions** (lines 52-59, 100-142, 228-247, 250-267): Already use `CREATE OR REPLACE FUNCTION` — safe to re-run
7. **ALTER TABLE ADD COLUMN** (lines 149-162): Already use `IF NOT EXISTS` — safe to re-run
8. **Backfill DO block** (lines 274-294): Uses `WHERE user_id IS NULL` — safe to re-run (no-ops if already done)
9. **View** (lines 300-320): Uses `DROP VIEW IF EXISTS` + `CREATE OR REPLACE VIEW` — safe to re-run

**Acceptance Criteria:**
- [ ] `user_profiles` table exists with RLS
- [ ] `handle_new_user` trigger fires on new signups
- [ ] `email_filters` table exists with RLS
- [ ] `user_id` column exists on `applications`, `raw_emails`, `pipeline_runs`, `pipeline_tasks`
- [ ] All 11 RLS policies active (verified via `pg_policies` query)
- [ ] Existing data backfilled to first auth.users entry
- [ ] `application_stats` view is per-user with `security_invoker = true`
- [ ] Migration can be re-run without errors (idempotent)

**Blocked By:** Nothing (foundation task)
**Blocks:** T-002, T-003, T-004, T-005, all EPIC 3 tasks
